How it Works
Three steps. Zero dependencies.

Write Markdown like you already do. md2cast handles the rest.

1

Write Markdown

Use standard Markdown with code blocks. No special syntax required.

# My Tutorial

Run this command:

```bash
echo "Hello, world!"
```
2

Run md2cast

One command. Generates a .cast file, GIF, or a full HTML page.

$ md2cast tutorial.md

$ md2cast tutorial.md --gif

$ md2cast tutorial.md --render

$ md2cast tutorial.md --render-html
3

Share Anywhere

Play in terminal, embed in docs, upload to asciinema.org, or convert to GIF.

# Play in terminal
$ asciinema play demo.cast

# Open HTML page
$ open demo.html

# Embed GIF in README
![demo](assets/demo.gif)

Features
Everything you need.
Nothing you don't.
⚙

Execute Mode

Actually run bash commands and capture real output. No more fake screenshots.

🎨

Themes

Custom colors, fonts, timing, and terminal size via JSON config. Auto-discovers theme files.

📸

Render Mode

Generate new Markdown with GIF screencasts embedded above each code block. Docs that move.

🌐

HTML Export

Self-contained HTML page with interactive asciinema players. One file, zero server needed.

✍

Directives

Fine-tune with HTML comments. Pause, skip, change prompt, control typing speed — per block.

🌈

Syntax Highlighting

Python, YAML, JSON, and more — automatically colored with Pygments. Any theme you want.

✂

Split Mode

One .cast per section. Perfect for multi-step tutorials with individual playback.

🛠

Zero Dependencies

Python 3.6+. No npm, no Docker, no build step. Single file, runs anywhere.

📈

Smart Sizing

Auto-sizes terminal rows to content. Short blocks stay compact, no wasted space.
